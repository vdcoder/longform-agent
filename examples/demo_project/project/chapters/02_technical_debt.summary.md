Chapter 2 defines technical debt using Ward Cunningham's original framing and
distinguishes three types: deliberate (conscious shortcut with a payback plan),
inadvertent (code that aged poorly), and bit rot (patterns inconsistent with
the modern system). It recommends making debt visible through friction logs,
change failure rate tracking, and time-to-first-test measurements. For
reduction, it advocates the boy scout rule backed by a protected 20% sprint
budget for technical work, and a product-style proposal process for large
architectural changes. The chapter emphasises weaving debt reduction into
normal work rather than treating it as a separate initiative.
